Polymarket RBI Bot

Research · Backtest · Incubate — Documentation complete du systeme de trading automatise

Introduction

Le Polymarket RBI Bot est un systeme de trading automatise qui execute trois strategies independantes sur la plateforme Polymarket (marche de prediction basee sur Polygon).

Le systeme suit la methodologie RBI :

de strategies

Etape 2

donnees historiques

Chaque bot fonctionne de maniere autonome avec son propre compte Polymarket, sa propre strategie et sa propre gestion de position.

Installation

Prerequis

  • Python 3.10+ installe et dans le PATH
  • pip (inclus avec Python)
  • Un wallet Polygon avec des fonds pour trader en live

Installation automatique

Double-cliquer sur start.bat — il s'occupe de tout :

# Ce que fait start.bat automatiquement :
1. Verifie que Python est installe
2. Cree un environnement virtuel (.venv)
3. Installe toutes les dependances (requirements.txt)
4. Copie .env.example → .env si manquant
5. Lance le serveur sur http://localhost:1818
6. Ouvre le dashboard dans le navigateur

Installation manuelle

# Creer et activer le venv
cd "C:\DEV POWERSHELL\TRADING BOT"
python -m venv .venv
.venv\Scripts\activate

# Installer les dependances
pip install -r requirements.txt

# Configurer l'environnement
copy .env.example .env
# Editer .env avec tes cles

# Lancer
python api/server.py

Strategie 1 : MACD (3/15/3)

Type

Momentum / Trend Following

Signal d'entree

Croisement de la ligne MACD au-dessus de la ligne signal

Signal de sortie

Croisement inverse ou stop-loss / take-profit

La strategie MACD (Moving Average Convergence Divergence) detecte les changements de momentum. Les parametres rapides (3/15/3) sont optimises pour les mouvements courts sur des fenetres de 5 minutes.

Comment ca marche

  1. Calcul de la ligne MACD = EMA(3) - EMA(15) des prix de cloture
  2. Calcul de la ligne signal = EMA(3) de la ligne MACD
  3. L'histogramme = difference entre les deux lignes
  4. BUY quand la ligne MACD croise au-dessus du signal (bullish crossover)
  5. SELL quand la ligne MACD croise en dessous du signal (bearish crossover)
Ideal pour : Les marches en tendance avec du volume. Fonctionne mal en range (marche lateral) — genere des faux signaux.

Strategie 2 : RSI + VWAP

Type

Mean Reversion / Counter-Trend

Signal d'entree

RSI < 30 (survendu) ET prix sous le VWAP

Signal de sortie

RSI > 50 et prix au VWAP, ou RSI > 70

Cette strategie parie sur le retour a la moyenne. Quand tout le monde panique et vend (RSI tres bas), elle achete en anticipant un rebond vers la "juste valeur" (VWAP).

Indicateurs utilises

  • RSI (14) — Relative Strength Index. Mesure si un actif est survendu (<30) ou surachete (>70)
  • VWAP — Volume Weighted Average Price. Prix moyen pondere par le volume = "prix juste" du marche
Ideal pour : Les pullbacks apres des mouvements violents. Quand le marche surreagit a une nouvelle.

Strategie 3 : CVD Divergence

Type

Volume Delta / Reversal Detection

Signal d'entree

Divergence entre la direction du prix et le CVD

Signal de sortie

Convergence prix/CVD ou stop-loss / take-profit

La strategie CVD analyse la pression d'achat vs de vente reelle cachee dans le volume. Quand le prix dit une chose mais que l'argent fait le contraire, elle suit l'argent.

Les deux signaux cles

DivergencePrixCVDSignalInterpretation
BullishBaisse ↓Monte ↑BUYAchat cache — les gros achetent pendant que les petits paniquent
BearishMonte ↑Baisse ↓SELLVente cachee — les gros vendent pendant que les petits FOMO
Ideal pour : Detecter les retournements de tendance. Fonctionne bien quand il y a du volume et de la volatilite.

Backtesting

Le backtesting permet de tester une strategie sur des donnees historiques avant de risquer de l'argent reel.

# Lancer le backtest des 3 strategies
python deploy/run_backtest.py

# Ce que ca fait :
# 1. Telecharge 30 jours de candles BTC/USDT (Binance)
# 2. Execute chaque strategie sur les donnees
# 3. Calcule les metriques de performance
# 4. Affiche un classement

Metriques calculees

MetriqueDescriptionSeuil minimum
Win RatePourcentage de trades gagnants> 55%
Profit FactorGains bruts / Pertes brutes> 1.5
Max DrawdownPire perte depuis un sommet< 20%
Sharpe RatioRendement ajuste au risque (annualise)> 1.0
Nombre de tradesTaille de l'echantillon> 100
Attention : Un backtest n'est PAS une garantie. L'overfitting (optimiser pour le passe) est le piege #1. Un backtest parfait peut echouer en live a cause du slippage, des frais, et de la liquidite.

Risk Manager

Le Risk Manager est la derniere ligne de defense avant chaque trade. Aucun ordre ne passe sans son approbation.

Regles appliquees

RegleCondition de blocageConfigurable
Taille maxPosition > MAX_POSITION_SIZE.env
Positions maxPositions ouvertes >= MAX_OPEN_POSITIONS.env
Perte journalierePnL du jour <= -MAX_DAILY_LOSS.env
Stop-LossPerte sur une position > 5%config/settings.py
Take-ProfitGain sur une position > 10%config/settings.py

Si la perte journaliere est atteinte, tous les bots s'arretent jusqu'au lendemain (reset a minuit UTC).

Dashboard

Le dashboard web (accessible sur http://localhost:1818) permet de :

  • Demarrer / arreter chaque bot individuellement avec les toggles
  • Voir les metriques en temps reel : PnL, win rate, Sharpe ratio
  • Suivre l'equity curve (evolution du capital) de chaque strategie
  • Consulter le trade log avec les 50 derniers trades
  • Surveiller le Risk Manager : perte journaliere, positions ouvertes
  • Kill All Bots en urgence (bouton rouge)
  • Modifier les settings : taille, stop-loss, take-profit, mode dry run

Le dashboard interroge l'API toutes les 3 secondes pour des donnees live.

Settings via API

# Exemple : modifier la taille de position
curl -X PUT http://localhost:1818/api/settings \
  -H "Content-Type: application/json" \
  -d '{"position_size": 5.0, "dry_run": true}'

Avertissement

AVERTISSEMENT IMPORTANT

Ce projet est fourni a des fins educatives et experimentales. Le trading comporte des risques de perte financiere. Les performances passees (backtests) ne garantissent pas les resultats futurs.

Regles d'or :
• Ne jamais investir plus que ce que tu peux te permettre de perdre
• Toujours commencer en mode DRY_RUN
• Respecter l'echelle d'incubation ($1 → $5 → $10...)
• Surveiller les bots regulierement
• 99% des traders manuels perdent de l'argent — les bots ne sont pas magiques
