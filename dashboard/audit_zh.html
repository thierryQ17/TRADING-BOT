严重 阻止实盘交易 3 个问题
1
Bot 管理器 = 随机模拟器
api/bot_manager.py:144-168
_run() 循环使用 random.random() 生成随机交易。Trader 被实例化但从未使用。实盘模式下不会执行任何真实交易 — 仪表板显示的是虚假数据。
→ 在 _run() 循环中接入 Trader.execute_once()，通过 Polymarket API 或 ccxt 获取真实数据。将模拟模式保留为明确的备用方案。
2
硬编码的 Token ID
api/bot_manager.py:135
token_id="placeholder" 被硬编码。没有选择真实 Polymarket 市场的机制。即使 Trader 被接入，也不知道在哪个市场操作。
→ 在仪表板和 API 中添加市场选择器（POST /api/bots/{key}/start 接受 token_id 参数）。通过 Polymarket API 提供活跃市场列表。
3
无实时数据
bot/trader.py:106-122
run_loop() 遍历传入的静态 DataFrame。没有实时数据获取 — 循环逐行重放历史数据而不是消费实时数据流。
→ 在 run_loop() 中实现周期性获取：每次迭代调用 downloader.fetch_ohlcv() 或 polymarket_client.get_price()，配合 time.sleep(interval)。以增量方式构建 DataFrame。
重要 可靠性受损 4 个问题
4
PnL 计算语义模糊
bot/position_tracker.py:52-55
pnl = (exit - entry) / entry * size — size 是美元金额，所以结果是百分比乘以一个金额。数学上正确，但语义混乱：PnL 纯粹因为巧合才是美元。
→ 明确为 pnl = (exit - entry) * quantity 或 quantity = size / entry（数量是单位数）。添加注释说明结果单位。
5
每次交易都调用 init_db()
data/storage.py:66-67
每次调用 log_trade() 都会执行 init_db()，创建新的 SQLite 连接并重新执行 CREATE TABLE IF NOT EXISTS。没有连接池 — 高频交易时存在 database is locked 风险。
→ 在启动时只初始化一次数据库。使用 threading.Lock 的单例连接或改用 sqlite3.connect(check_same_thread=False) 配合连接池。
6
PUT /api/settings 无验证
api/server.py:114-117
该端点接受任意值：position_size: -999、stop_loss_pct: 500。Pydantic 模型 SettingsUpdate 定义了类型但没有边界。恶意调用或 UI 错误可能破坏风险管理器。
→ 添加 Pydantic 验证器：position_size 用 Field(ge=0.01, le=1000)，百分比用 Field(ge=0.1, le=50)。超范围返回 422。
7
全局 DRY_RUN 突变
api/bot_manager.py:259
settings.DRY_RUN = self._settings["dry_run"] 从线程修改模块级全局变量。所有机器人线程共享此状态而无同步 — 如果机器人正在执行，则存在竞态条件。
→ 将 dry_run 作为实例参数传递给 Trader 和 PolymarketClient。删除全局变更。每个机器人读取自己的配置。
中等 设计和健壮性 5 个问题
8
CORS 完全开放
api/server.py:20-25
allow_origins=["*"] 允许任何网站调用 API。在生产环境中，内存中有 Polygon 私钥，恶意网站可能通过用户浏览器触发交易。
→ 限制为 allow_origins=["http://localhost:1818"] 或通过 .env 设为可配置。
9
Scaler 无下降机制
incubation/scaler.py
递进比例从 $1 上升到 $100（满足条件时），但永远不会下降。如果升级后表现恶化，机器人继续以大仓位交易。
→ 添加 _evaluate_level_down()：如果胜率跌破 45% 或利润系数跌破 0.8，则下降一级。保护资本。
10
仅在收盘价上检查止损
backtesting/engine.py:48-83
回测引擎在蜡烛的 close 上检查止损，而非 low（BUY）或 high（SELL）。实际上止损会在蜡烛内触发 — 回测因此过于乐观。
→ 在 BUY 仓位的 curr["low"] 和 SELL 仓位的 curr["high"] 上检查止损。用止损价格而非收盘价作为出场价格。
11
CVD = 启发式近似
strategies/cvd_strategy.py:24-28
成交量 Delta 由 volume * (close - open) / (high - low) 近似。在没有 tick 级数据的情况下这是合理的启发式，但信号置信度应反映这种不确定性。
→ 对置信度应用折扣系数（如 confidence *= 0.7）以补偿近似误差。在策略中记录该限制。
12
夏普比率年化错误
api/bot_manager.py:215
arr.mean() / arr.std() * (252 ** 0.5) 使用 252（交易日）作为年化因子。但模拟中每 2-4 秒就有一笔交易 — 年化不一致，夏普比率被高估。
→ 根据实际交易频率计算，使用 sqrt(N) 其中 N 是每年的周期数。或显示非年化夏普并标注"每笔交易"。
轻微 代码质量 3 个问题
13
无集成测试
tests/
单元测试隔离覆盖了策略、回测和风险管理器。但没有针对 FastAPI 路由、BotManager 线程或完整流程 signal → risk → order → position 的测试。
→ 使用 FastAPI 的 TestClient 添加端点测试。使用 DRY_RUN 模式下模拟的 PolymarketClient 测试完整流程。
14
函数内延迟导入
api/bot_manager.py:146, 213
import random 和 import numpy 在频繁调用的函数体内导入。开销可以忽略（Python 缓存模块），但这是降低可读性的坏习惯。
→ 将导入移到文件顶部，与其他模块导入放在一起。
15
无优雅关闭
api/bot_manager.py
服务器停止时（Ctrl+C），守护线程无清理即终止。未保存未平仓仓位，未在 Polymarket 上取消挂单。
→ 拦截 SIGINT/SIGTERM 以调用 kill_all() + 将状态保存到 SQLite。使用 FastAPI 的 shutdown 事件。
