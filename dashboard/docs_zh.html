Polymarket RBI 交易机器人

研究 · 回测 · 孵化 — 自动化交易系统完整文档

简介

Polymarket RBI Bot 是一个自动化交易系统，在 Polymarket（基于 Polygon 的预测市场）上执行三种独立策略。

系统遵循 RBI 方法论：

第 1 步
Research
寻找策略思路
→
第 2 步
Backtest
用历史数据测试
→
第 3 步
Incubate
以最小风险部署

每个机器人独立运行，有自己的 Polymarket 账户、自己的策略和自己的仓位管理。

安装

前提条件

  • Python 3.10+ 已安装并在 PATH 中
  • pip（随 Python 一起安装）
  • 一个Polygon 钱包，有资金用于实盘交易

自动安装

双击 start.bat — 它会自动完成所有步骤：

# start.bat 自动执行：
1. 检查 Python 是否安装
2. 创建虚拟环境（.venv）
3. 安装所有依赖（requirements.txt）
4. 如果缺失则复制 .env.example → .env
5. 在 http://localhost:1818 启动服务器
6. 在浏览器中打开仪表板

手动安装

# 创建并激活虚拟环境
cd "C:\DEV POWERSHELL\TRADING BOT"
python -m venv .venv
.venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 配置环境
copy .env.example .env
# 用你的密钥编辑 .env

# 启动
python api/server.py

策略 1：MACD (3/15/3)

类型

动量 / 趋势追踪

入场信号

MACD 线在信号线上方金叉

出场信号

死叉或止损 / 止盈

MACD（移动平均收敛发散）检测动量变化。快速参数（3/15/3）针对 5 分钟 K 线上的短期波动进行了优化。

工作原理

  1. 计算MACD 线 = 收盘价的 EMA(3) - EMA(15)
  2. 计算信号线 = MACD 线的 EMA(3)
  3. 直方图 = 两线之差
  4. BUY — MACD 线在信号线上方金叉时
  5. SELL — MACD 线在信号线下方死叉时
适合： 有明显趋势和成交量的品种。在盘整市场（横盘）表现不佳 — 会产生假信号。

策略 2：RSI + VWAP

类型

均值回归 / 逆势交易

入场信号

RSI < 30（超卖）且价格在 VWAP 下方

出场信号

RSI > 50 且价格在 VWAP，或 RSI > 70

此策略押注于均值回归。当所有人恐慌抛售时（RSI 极低），它买入，期待反弹至"公允价值"（VWAP）。

使用的指标

  • RSI (14) — 相对强弱指数。衡量资产是否超卖（<30）或超买（>70）
  • VWAP — 成交量加权平均价。成交量加权的平均价格 = 市场的"公允价值"
适合： 剧烈波动后的反弹。当市场对新闻反应过度时。

策略 3：CVD 背离

类型

成交量 Delta / 反转检测

入场信号

价格方向与 CVD 的背离

出场信号

价格与 CVD 收敛或止损 / 止盈

CVD 策略分析隐藏在成交量中的真实买卖压力。当价格说一套、资金流向说另一套时，它跟随资金。

两个关键信号

背离类型价格CVD信号解读
看涨背离下跌 ↓上涨 ↑BUY隐藏买入 — 大户在散户恐慌时买入
看跌背离上涨 ↑下跌 ↓SELL隐藏卖出 — 大户在散户 FOMO 时卖出
适合： 检测趋势反转。有成交量和波动性时表现良好。

回测

回测让你在历史数据上测试策略，然后再用真实资金冒险。

# 运行 3 种策略的回测
python deploy/run_backtest.py

# 执行流程：
# 1. 通过 ccxt 下载 BTC/USDT 30 天 K 线（Binance）
# 2. 在数据上运行每个策略
# 3. 计算性能指标
# 4. 显示排名

计算的指标

指标描述最低阈值
胜率盈利交易占比> 55%
盈利因子总盈利 / 总亏损> 1.5
最大回撤从峰值到谷底的最大损失< 20%
夏普比率风险调整后收益（年化）> 1.0
交易数量样本大小> 100
注意： 回测不是保证。过度拟合（为过去优化）是第一大陷阱。完美的回测可能在实盘中因滑点、费用和流动性而失败。

风险管理器

风险管理器是每笔交易前的最后一道防线。未经其批准，任何订单都不会执行。

执行的规则

规则阻止条件可配置
最大仓位仓位 > MAX_POSITION_SIZE.env
最大持仓开仓数 >= MAX_OPEN_POSITIONS.env
日亏损.env
止损单笔亏损 > 5%config/settings.py
止盈单笔盈利 > 10%config/settings.py

如果达到日亏损限额，所有机器人停止直到次日（UTC 午夜重置）。

仪表板

Web 仪表板（访问 http://localhost:1818）可以：

  • 通过开关单独启动/停止每个机器人
  • 查看实时指标：盈亏、胜率、夏普比率
  • 追踪每种策略的资金曲线（资金变化）
  • 查看交易日志（最近 50 笔交易）
  • 监控风险管理器：日亏损、开仓情况
  • 紧急情况点击红色全部停止按钮
  • 修改设置：仓位、止损、止盈、模拟模式

仪表板每3 秒查询一次 API 获取实时数据。

API 端点

REST API 访问地址 http://localhost:1818/api/。交互式 Swagger 文档在 /docs。

方法端点描述
GET/api/bots3 个机器人的列表，含状态、盈亏、胜率
POST/api/bots/{key}/start启动一个机器人（macd、rsi、cvd）
POST/api/bots/{key}/stop停止一个机器人
POST/api/bots/kill-all停止所有机器人
GET/api/metrics全局指标（总盈亏、夏普等）
GET/api/trades?limit=50最近的交易
GET/api/risk风险管理器状态
GET/api/settings当前设置
PUT/api/settings修改设置

通过 API 修改设置

# 示例：修改仓位大小
curl -X PUT http://localhost:1818/api/settings \
  -H "Content-Type: application/json" \
  -d '{"position_size": 5.0, "dry_run": true}'

已知风险

风险描述缓解措施
过度拟合为过去优化的策略在实盘中失败在不同时间段回测，走前分析
滑点实际成交价与预期不同仅使用限价单（Polymarket 0 手续费）
流动性订单无法成交或只能部分成交小仓位，活跃品种
API 宕机 Polymarket 在持仓期间不可访问内置止损，持续监控
密钥泄露私钥暴露 = 钱包清空.gitignore 包含 .env，永不提交
监管风险Polymarket 有地区限制确认你所在国家/地区的合法性

重要声明

重要警告

本项目仅供教育和实验目的。交易涉及财务损失风险。过去的表现（回测结果）不保证未来结果。

铁律：
• 永远不要投入你无法承受损失的资金
• 始终从 DRY_RUN 模式开始
• 遵循孵化缩放路线（$1 → $5 → $10...）
• 定期监控机器人
• 99% 的手动交易者亏损 — 机器人不是魔法
